### Задание 1: Анализ допустимости выражений (Task1.java)
**Файл:** `Task1.java`, класс `F`

**Вопрос:** Какие из предложенных выражений являются допустимыми?

**Ответ:**
- ✅ **Допустимые:** 1, 2, 3, 4, 6, 8
- ❌ **Недопустимые:** 5, 7

**Пояснения:**
1. `f.i` - допустимо (доступ к переменной экземпляра через объект)
2. `f.s` - допустимо, но не рекомендуется (доступ к статической переменной через объект)
3. `f.imethod()` - допустимо (вызов метода экземпляра через объект)
4. `f.smethod()` - допустимо, но не рекомендуется (вызов статического метода через объект)
5. `F.i` - **НЕ допустимо** (нельзя получить доступ к переменной экземпляра через класс)
6. `F.s` - допустимо (правильный доступ к статической переменной)
7. `F.imethod()` - **НЕ допустимо** (нельзя вызвать метод экземпляра через класс)
8. `F.smethod()` - допустимо (правильный вызов статического метода)

### Задание 2: Использование ключевого слова static (Task2.java)
**Файл:** `Task2.java`

**Правила:**
- `main()` должен быть static (точка входа в программу)
- `getCount()` НЕ должен быть static (обращается к переменной экземпляра)

## Основные концепции

1. **Статические члены класса**
   - Принадлежат классу, а не объекту
   - Доступны через имя класса
   - Могут быть вызваны без создания объекта

2. **Члены экземпляра**
   - Принадлежат объекту
   - Требуют создания экземпляра класса
   - Не доступны через имя класса

3. **Массивы объектов**
   - Требуют инициализации элементов
   - Обращение к методам возможно только у инициализированных объектов

4. **Сортировка методом выбора**
   - Сложность: O(n²)
   - Находит минимальный элемент и перемещает его в начало
